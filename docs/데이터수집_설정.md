# 데이터 수집 자동화 — 설정 안내

매체별 광고 데이터를 GitHub Actions가 매일 자동으로 BigQuery에 적재한다.
이 문서는 **무엇이 어디서 어떻게 수집되는지 + 필요한 GitHub Secrets 목록**을 정리한다.

> ⚠️ **보안 규칙**: 이 레포는 공개(public)다. **토큰·시크릿·앱 비밀번호 등 "값"은 이 파일에도, 코드에도 절대 넣지 말 것.**
> 값은 **GitHub → Settings → Secrets and variables → Actions** 에만 등록한다. (아래 표엔 Secret "이름"과 비밀이 아닌 식별자만 적는다.)

## 4. 인증값 발급 방법 (요약)

- **메타**: Meta 앱 → 마케팅 API → 도구 → `ads_read` 체크 → 토큰 받기(60일).
  무기한 자동화하려면 비즈니스 관리자에서 **시스템 사용자 토큰** 발급 권장.
- **카카오모먼트**: 공식대행사(지아이오엠)가 **[앱]→[추가 기능 신청]→카카오모먼트** 로 광고계정 `669973` 권한 신청 → 카카오 승인 후,
  **비즈니스 인증** OAuth로 **비즈니스 토큰(리프레시)** 발급. (일반 카카오 로그인 토큰으로는 `permission denied -5`)
- **모비온**: 모비온 **광고주센터/admin(admin.mobon.net)** 에서 **Access Token** 발급.
  첫 API 호출 IP는 자동 화이트리스트 등록. 호출 한도 120건/분. 광고주센터 API로 **일간 리포트** 제공.

## 5. 새 매체 추가 절차 (개발 메모)

1. `collect_<매체>.py` 작성 — 인증은 전부 `os.environ`(Secrets)에서 읽기. BigQuery는 **DML 금지**,
   "읽고→해당 media 제거→합쳐→WRITE_TRUNCATE" 멱등 방식(`collect_meta.py`·`collect_google.py` 참고).
2. `ad_etc`(기타매체) 또는 `ad_keyword`에 `media` 태깅해 적재.
3. `.github/workflows/collect_<매체>.yml` — 매일 스케줄 + `concurrency: bq-ad-etc-write`(같은 표 동시쓰기 방지).
4. 대시보드 `load_etc()`가 해당 media를 읽는지 확인(시트 수기 → 자동수집 전환 시 시트 media_list에서 제외).
