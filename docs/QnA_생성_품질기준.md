# QnA 자동생성 품질 기준서 (카테고리 정합 + GEO)

> 목적: QnA 원고 생성 프롬프트를 고치기 전에 **① 카테고리 전체 ② 카테고리별 적합/금지 내용 ③ GEO 최적 글 형식**을 확정한다.
> 배경: 외부 피드백(강동현) — (1) "처벌과 대응은?" 류 템플릿 질문 금지, (2) **민사 카테고리에 형사식 '처벌' 질문이 섞임(카테고리 불일치)**, (3) 챗지피티·제미나이·퍼플렉시티 기준 **GEO 품질 낮음** → 프롬프트 개편 필요.
> 이 문서를 확정한 뒤 생성 프롬프트에 규칙으로 반영("AI 학습")한다.

- **처벌·형량·구속·기소·전과 = '형사' 개념.** ✅형사·성범죄·금융범죄 / △학폭·음주교통·소년·외국인(형사 요소 있을 때).

## 2. 카테고리별 — 적합 질문 프레임 vs 금지

| 카테고리 | ✅ 적합한 질문·내용 프레임 | ❌ 금지/부적합 |
|---|---|---|
| **형사** | 죄의 성립요건, 처벌 수위·형량 범위, 구속/불구속, 수사 절차, 합의·선처·감형, 전과·집행유예, 불송치·기소유예 | 손해배상 청구(→민사), 이혼·상속(→가사) |
| **성범죄** | 성립요건, 처벌·신상등록·취업제한, 무고 대응, 합의·피해자 진술, 2차피해 | 단순 민사 손배만 다루기, 타 죄종 |
| **학교폭력** | 학폭위 처분 종류(1~9호)·불복(행정심판/소송), 가해 시 형사, 피해 시 민사 손배, 생기부 기재 | "처벌 수위"를 학폭위 사안에 형사처럼 쓰기 |
| **음주운전·교통사고** | 음주 처벌·수치별 기준, 면허취소·정지·구제(행정심판), 위드마크, 교통사고 형사(치사상)·합의·보험·손배 | 카테고리 밖 죄종 |
| **민사·행정** | 손해배상·채권추심·계약분쟁·명예훼손 손배, 행정처분 취소·집행정지, 입증·소송절차·비용 | **처벌·형량·구속(→형사)**, 이혼(→가사) |
| **이혼·가사** | 이혼(협의/재판)·사유, 양육권·친권·양육비, 재산분할·위자료, 상속·유류분, 가정보호 | **처벌·형량**, 형사 프레임 |
| **소년범죄** | 소년보호처분(1~10호), 소년부 송치 vs 형사, 소년 형사특칙, 전과·보호관찰, 부모 역할 | 성인 형사처럼 "처벌 수위"만 강조 |
| **행정소송** | 행정처분 취소·무효, 집행정지, 영업정지·인허가·과징금 불복, 정보공개, 소송요건·제소기간 | **형벌·구속(→형사)** |
| **금융범죄** | 사기·횡령·배임·보이스피싱·투자사기 성립·처벌, **피해자 환급·구제·민사 회수** 병행 | 피해구제 없이 처벌만, 무관 죄종 |
| **건설·부동산분쟁** | 공사대금·하자보수·유치권·명도·분양·재건축/재개발·임대차 분쟁, 청구·입증·소송·비용 | **처벌(→형사)**, 불법건축 형사화 |
| **소액및손해배상** | 소액소송(3천만 이하)·지급명령·손해배상 청구, 절차·기간·비용·집행 | **처벌**, 형사 프레임 |
| **회생·파산** | 개인회생·개인파산·면책, 채무조정·변제계획, 자격/요건·비용, 압류/추심 중단 | **처벌**(사해행위·재산은닉 형사화는 별개 취급) |
| **외국인·출입국** | 비자·체류자격·연장·변경, 강제퇴거·출국명령 대응, 난민·귀화, 출입국사범(형사 요소) | 카테고리 밖 일반 형사 |

> 공통: 카테고리에 **없는 개념을 억지로 넣지 않는다.** 특히 '처벌' 금지 카테고리(민사·행정·이혼가사·건설부동산·소액손배·회생파산·행정소송)에는 처벌·형량·구속 단어 자체를 쓰지 않는다.

## 3. 뻔한 템플릿 질문 금지

- ❌ "○○의 처벌과 대응은?", "○○란 무엇인가?", "○○ 총정리" 같은 **일반형 템플릿**.
- ✅ **실수요 키워드 기반 구체 질문**(사람이 실제 AI/검색에 묻는 자연어):
  - 예) "음주 0.08%면 초범도 실형인가요?", "전세보증금 안 돌려줄 때 지급명령부터 해도 되나요?", "학폭위 3호 처분 생기부에 남나요?"
- 질문은 **상황·수치·조건이 담긴 문장**으로. 한 글=한 질문=한 직답.

## 4. GEO 최적 글 형식 (Generative Engine Optimization)

> GEO = 챗지피티·제미나이·퍼플렉시티·AI 개요 등 **생성형 엔진이 인용·노출하기 좋게** 쓰는 것. SEO의 AI판.
> (여기서 GEO는 '지역(geographic)'이 아니라 생성엔진 최적화. 단, 지역+법률 결합 키워드는 별도 유입 레버로 유효.)

### 원칙 (LLM이 잘 물어가는 글의 공통점)
1. **직답 우선(Answer-first / 역피라미드)** — 질문 바로 뒤 **1~3문장 결론**부터. 근거·부연은 그 다음.
2. **질문형 소제목 + 구조화** — 자연어 질문을 H2/H3 소제목으로. 리스트·표·단계로 쪼갠다. (FAQ 구조가 이상적)
3. **인용가능한 조각(citable chunks)** — 각 문단이 **그 자체로 하나의 답**이 되게. LLM은 문단 단위로 잘라 인용한다.
4. **구체성·근거** — 정확한 **법령·조문명**(예: 도로교통법 제148조의2), 형량/기간/비용의 **대략 수치·범위**, 요건 열거. 근거 있는 글이 인용된다.
5. **개체·용어 정확성(엔티티)** — 정확한 죄명·법령명·기관명·절차명. 모호한 일반론 금지.
6. **신뢰·최신성(E-E-A-T)** — 작성 주체(법무법인/변호사), **개정법 반영 시점/날짜**, 필요 시 출처. "일반적 정보이며 사안별로 다름" 고지.
7. **자연어 질의 매칭** — 실제 사용자가 AI에 묻는 대화체 질문에 정확히 대응.
8. **간결·명료** — 군더더기·중복 제거. 한 질문 한 답, 과장·홍보문구 최소.

### 우리 게시판 글 골격 (권장 템플릿)
```
[제목] 자연어 구체 질문 (지역/조건 포함 가능)
─ 핵심 요약 답변(summary 필드): 결론 2~3문장 (직답)
─ 상세 답변(wr_content):
   • 소제목1(질문 세분화) → 직답 + 근거(조문/요건)
   • 소제목2 → 절차·단계(번호 리스트)
   • 소제목3 → 주의점·예외·비용/기간(수치)
   • 마무리: 사안별 상이 고지 + (법인 관점 1문장)
─ 태그(wr_4): 죄명/절차/지역 등 구체 키워드
```
> 게시판 스킨이 '핵심요약/상세' 골격을 스스로 그리므로, 업로드 시엔 값만 넣는다(현행 `qna_upload` 그대로).

## 5. 다음 단계 (이 기준을 "AI에 학습")

1. **생성 프롬프트에 규칙 주입**: (a) 카테고리별 적합/금지 프레임(§2 표), (b) '처벌 금지 카테고리' 하드룰, (c) 템플릿 질문 금지(§3), (d) GEO 골격(§4)을 시스템 프롬프트/생성 규칙으로 인코딩.
2. **카테고리별 few-shot 예시**: 각 카테고리 '좋은 질문·답' 1~2개를 예시로 넣어 형태를 고정.
3. **자체 검증 게이트**: 생성 후 (a) 카테고리-금지어(처벌 등) 검사, (b) 템플릿 질문 검사, (c) 조문/근거 포함 여부 검사 → 미달이면 재생성.
4. **강동현 GEO 기준 확보**: 그쪽이 'GEO 점수'를 **구체적으로 무엇으로 측정**하는지(어떤 항목·툴)를 받으면, 그 항목에 맞춰 §4를 더 정밀화.
